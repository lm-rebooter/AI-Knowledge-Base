# Architecture

## 分层职责

- `apps/web` 负责页面、交互、表单和用户体验。
- `apps/api` 负责认证、业务规则、数据库和调用 AI 服务。
- `apps/ai-service` 负责文档处理、向量化、检索和问答。

## 请求链路

1. 用户在 `web` 发起请求。
2. `api` 完成鉴权、校验和业务编排。
3. 当请求涉及 RAG 时，`api` 调用 `ai-service`。
4. `ai-service` 执行检索与生成，再把结果返回给 `api` 和 `web`。

## 为什么适合练手

这套结构能够让你明确区分：

- BFF 或主业务后端负责什么
- AI 服务应该隔离哪些职责
- 数据库存储和向量库存储为什么不应该混在一起
